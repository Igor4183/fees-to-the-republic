Это интерактивная задача.

Есть скрытая перестановка∗∗ 𝑝p длины 𝑛n. Вы можете взаимодействовать с ней, задавая следующий запрос не более чем 2𝑛2n раз:

  • Выберите два целых числа 𝑖i и 𝑥x такие, что 1≤𝑖≤𝒏−11≤i≤n−1 и 1≤𝑥≤1091≤x≤109. Жюри ответит 𝟶0, если 𝑝𝑖&𝑥pi&x†† равно нулю, и 𝟷1 в противном случае.

Важно: Вы не можете задавать запросы, касающиеся последнего элемента 𝑝𝑛pn (поскольку 𝑖≤𝑛−1i≤n−1).

Ваша цель — определить значение последнего элемента перестановки, 𝑝𝑛pn, используя не более чем 2𝑛2n запросов.

Обратите внимание, что интерактор не адаптивен. Это означает, что скрытая перестановка 𝑝p фиксирована с самого начала и не изменится в зависимости от ваших запросов.

∗∗Перестановкой длины 𝑛n является массив, состоящий из 𝑛n различных целых чисел от 11 до 𝑛n в произвольном порядке. Например, [2,3,1,5,4][2,3,1,5,4] — перестановка, но [1,2,2][1,2,2] не перестановка (22 встречается в массиве дважды) и [1,3,4][1,3,4] тоже не перестановка (𝑛=3n=3, но в массиве встречается 44).

††&& обозначает операцию побитового И.

Входные данные

Каждый тест состоит из нескольких наборов входных данных. В первой строке находится одно целое число 𝑡t (1≤𝑡≤1031≤t≤103) — количество наборов входных данных. Далее следует описание наборов входных данных.

Первая строка каждого набора входных данных содержит одно целое число 𝑛n (2≤𝑛≤2⋅1042≤n≤2⋅104) — длина перестановки 𝑝p.

Для каждого набора входных данных, после чтения 𝑛n, вы должны начать взаимодействие и найти ответ, прежде чем переходить к следующему набору входных данных.

Гарантируется, что сумма значений 𝑛n по всем наборам входных данных не превосходит 2⋅1042⋅104.

Протокол взаимодействия

Чтобы сделать запрос, выведите одну строку в следующем формате:

  • ?𝑖𝑥?ix (1≤𝑖≤𝑛−11≤i≤n−1, 1≤𝑥≤1091≤x≤109) — индекс для запроса и значение, с которым нужно выполнить побитовую операцию И.

После каждого запроса вы должны прочитать одно целое число 𝑏b:

  • 𝑏=𝟶b=0, если 𝑝𝑖&𝑥=0pi&x=0,
  • 𝑏=𝟷b=1 в противном случае.

Вы можете сделать не более чем 2𝑛2n таких запросов в каждом наборе входных данных.

Чтобы дать окончательный ответ, выведите одну строку в следующем формате:

  • !𝑥!x (1≤𝑥≤𝑛1≤x≤n) — значение 𝑝𝑛pn.

Взломы

Чтобы совершить взлом, используйте следующий формат.

Первая строка должна содержать одно целое число 𝑡t (1≤𝑡≤1031≤t≤103) — количество наборов входных данных.

Вторая строка каждого набора входных данных должна содержать 𝑛n целых чисел 𝑝1,𝑝2,…,𝑝𝑛p1,p2,…,pn (1≤𝑝𝑖≤𝑛1≤pi≤n) — элементы перестановки 𝑝p.
